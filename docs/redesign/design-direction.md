# Stellise リデザイン設計図（2026-06-13）

実装は Mac（Xcode）で行う。これは方向確定済みの設計図。MacでこのリポをpullしてP0から着手する。

## 1. コンセプト

「夜は沈み込む静けさ、朝は天気とともに気持ちよく立ち上がる」AIアラーム。
紺（ネイビー）を基調に、夜は黒へ深く沈め、朝は天気で明るく変える。アクセントはラベンダーパープル。

- **夜（NightView）= 没入ダーク**：紺→黒グラデ＋発光＋星。中央に大きな円形（就寝〜起床の残り時間）。
- **朝・タスク（DayView）= 明るめ＋天気連動**：背景がその日の天気で変わる（iOS天気アプリ風）。タスク完了は **Tinder式カードスワイプ**。
- 全体に **3Dソフトイラスト**（月・星・天気・タスクアイコン）。

## 2. デザインシステム

### カラートークン（叩き台・Macで微調整 / Asset Catalogに Color Set で定義）

**ベース・夜**
- `night/top`    `#1B1B3A`（深い紺）
- `night/bottom` `#0A0A14`（ほぼ黒）
- 縦グラデ top→bottom。星パーティクル＋中央にsoft glow。

**ベース・朝（天気連動グラデ。上→下）** ※「紺を暗くしていった感じ」を保つため、晴れでも水色に振りすぎず上品な紺寄りブルー
- 晴れ Clear   `#3E5C96` → `#9FB8E0`
- 曇り Cloudy  `#4C5468` → `#8E99AE`
- 雨   Rain    `#33405A` → `#5E6B86`
- 夜明け前 Dawn `#2A2A4E` → `#5B4B7A`（紺×パープル）
- 夜寄り Night  = ベース・夜と同じ

**アクセント（パープル）**
- `accent/primary` `#8B7CF6`（ラベンダー・メインCTA、リング、中央＋ボタン）
- `accent/light`   `#B4A9FF`
- `accent/deep`    `#6C5CE0`

**セマンティック**
- テキスト: 暗背景=白／白0.8、明背景(晴れ等)=濃紺 `#1B1B3A`。`isBrightBackground` 判定は継続。
- 遅延・警告: `#FF6B6B`（赤は遅延時のみ）
- 成功/完了: パープル系で統一（蛍光グリーンは使わない／既存方針踏襲）
- カード: `.ultraThinMaterial`（ガラス）。明背景では白寄り、暗背景では黒寄りに自然に効く。

### タイポgrafi
- 時計: `system`, weight `.thin`〜`.ultraLight`, 88–100pt, `.monospacedDigit()`。`design: .rounded` も検討。
- 見出し: `.title` / `.title2` semibold。アクセントで **italic serif** を1箇所だけ効かせるのは可（参照1枚目の "Let's create a sleep routine" の雰囲気）。
- 本文: `.callout` / `.subheadline`、補足 `.caption`。

### 形状・レイアウト
- 角丸: カード 28、ピル/ボタン 全角丸（capsule）、小要素 16。
- **ボトムナビ**: フローティングのピル型。中央に大きい円形 **＋ボタン**（パープル）。タブ例: ホーム / 履歴 / (＋) / 統計 / 設定。`AI Assistant` タブを置くのも◎（Stelliseの強みと相性）。
- カードはガラス＋わずかな内側ハイライトで立体感。

### 3D・モーション
- 3Dアイコン: 月・星・太陽・雲・雨、タスク（出発/朝食/服薬…）をソフト3D調で。調達は §5。
- 発光: 夜の時計・中央円に soft glow（`shadow` で代替可）。
- 遷移: 夜⇄朝はクロスフェード＋背景色補間。

## 3. 画面別仕様

### NightView（夜・没入ダーク）
- 背景: `night` グラデ＋星＋中央 glow。
- 上部: 「おやすみなさい、◯◯さん」。
- 中央: **大きな円形**。リングの進捗＝就寝→起床。中に **「あと 6時間30分」＋起床時刻**（参照1枚目中央のタイマー）。タップでアラーム時刻ピッカー。
- 下部: 睡眠環境音カード（再生／音種Menu／タイマー）。タップ領域を今より大きく、暗所コントラスト確保。
- **OLEDブラックアウト実装**（現状 `NightView.swift:146-148` はコメントのみ）：伏せ検知で真っ黒オーバーレイ→省電力＋まぶしさ低減。

### DayView（朝・天気連動・Tinderタスク）
- 背景: `WeatherBackground`（§4-1）。3D天体レイヤーを重ねる。
- ヘッダー: 日付／出発時刻／天気／移動（既存 HeaderView を踏襲、明背景は濃紺文字）。
- 時計: 大時計（明背景→濃紺、暗背景→白）。
- **タスク = カードスタック（Tinderスワイプ）**（§4-2）。1枚ずつ大きめカード（3Dアイコン＋タスク名＋時刻）。右=完了、左=スキップ/後で。全完了で「準備完了」。
- AIの一言（"傘を持って" 等）をカード上 or ヘッダー下に。
- 既存の「AIは間違えることがあります」注記は残す。

### Onboarding（離脱対策込み）
- **プリパーミッション画面**を新設：マイク（睡眠音）・通知が「なぜ必要か」「処理は端末内で完結＝プライバシー安全」を3D月ビジュアルで先に説明 → その後に許可ダイアログ。現状 `WelcomeView.swift:47-53` のいきなり許可を廃止。
- 流れ: ようこそ → 価値/権限説明 → 名前 → 身体情報 → 起床時刻 → 移動手段。

### Sleep Insight / 統計
- 睡眠スコア円（参照3枚目「61 / 100」）、週リング、睡眠ステージグラフ（パープル〜ブルーのグラデ）。
- **配色はダーク基調で統一**（世界観を割らない。明るいカードを浮かせる）。

### AlarmRinging
- 全画面。夜→朝への色変化で「起こす」。起床ミッション継続。

### Settings
- ダークのカードリスト。既存項目を新トークンで整える。

## 4. 主要インタラクション

### 4-1. 天気連動背景（WeatherBackground）
- `enum WeatherCondition { clear, cloudy, rain, snow, dawn, night }` を `AppState.weather` から決定。
- 各 condition に LinearGradient（§2のパレット）＋3Dレイヤー（太陽/雲/雨粒）。
- 時間帯（早朝/日中/夕/夜）でも補間。`TimelineView` で雲の微動など軽アニメ。
- **段階実装**: まず静的グラデ4〜5種 → 後で3D天体・パーティクルのアニメを足す。

### 4-2. Tinderカードスワイプ（タスク完了）
- カードスタック表示。最前面カードに `DragGesture`。
  - `offset` に追従＋`rotationEffect`（端ほど傾く）＋ドラッグ量で薄く「完了」/「スキップ」ラベルを出す。
  - しきい値（画面幅の ~30%）超で確定、`withAnimation(.spring)` で飛ばす→次カードを前面へ。
  - 右=完了（`task.isCompleted = true`＋触覚 `UIImpactFeedbackGenerator`）、左=スキップ（末尾へ送る/今日はやらない）。
- fityouで使った「最初の数pxで軸判定（縦スクロールと横スワイプの両立）」の考え方を流用。SwiftUIなのでCSSは無関係だが、**カードスタックは独立ジェスチャ**にして親スクロールと競合させない。
- 既存の「左スワイプ完了（List, `DayView.swift:169`）」は廃止し、このカードUIに置き換え。

## 5. アセット調達（3D）
- 候補: ①フリー/購入の3Dアイコンセット（統一トーンのものを1セット） ②AI画像生成（プロンプトとライティングを固定して統一感を担保） ③Blender自作。
- **MVPは「グラデ＋内外シャドウの擬似3D」から**始め、本物の3Dは後差し替え可能なコンポーネント設計に。
- 月・星はまず既存アセット（`image-space-background` 等）を活かしつつ差し替え。

## 6. 実装フェーズ（おすすめ順＝効果×低コスト）
- **P0 カラー/トーン刷新**: Asset Catalogに新カラー、タイポ・角丸・ガラスを共通化し全画面へ適用。一番効いて低コスト。
- **P1 タスクのTinderカードスワイプ化**（DayView）。
- **P2 天気連動背景**（まずグラデ4〜5種、後で3D天体アニメ）。
- **P3 NightView円形リデザイン**（残り時間表示）＋OLEDブラックアウト実装。
- **P4 Sleep Insight/統計＋ボトムナビ刷新**（AI Assistantタブ）。
- **P5 オンボーディング＋プリパーミッション**。
