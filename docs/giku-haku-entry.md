# 技育博（サポーターズ）エントリー — Stellise

> 締切: 2026-06-30(火) / 開催: 2026-09-19(土) 虎ノ門ヒルズ ステーションタワー36F / 個人エントリー

## 作品名
Stellise（ステライズ）

## ひとことキャッチ
朝の「あと5分」からの遅刻をなくす、AIアラーム＆朝ルーティン最適化アプリ

## 作品概要
寝坊・電車遅延・急な天候変化など、予測しづらい朝のルーティン崩れを、オンデバイスAIと各種API連携で解決するiOSアプリです。端末内のTensorFlow Lite（YAMNet）とCoreMotionで睡眠の深さを推定し、眠りの浅いタイミングで起こすスマートアラームを実現。さらに天気・経路情報から出発時刻を算出し、遅延や悪天候を検知すると「何を削り何を先にやるか」をAIが判断してタスク配分を自動で組み直します。睡眠・音声解析は端末内で完結し、データを外部送信しないプライバシー重視設計です。

## 開発のきっかけ・込めた想い
きっかけは、自分自身が「あと5分」と二度寝しては遅刻し、起きられても天気や電車の遅れで予定が崩れて朝にバタバタしてしまう、という日常の小さなストレスでした。世の中の二度寝防止アプリは、パズルや計算で無理やり脳を起こすものが多く、「起こされる」体験そのものがストレスになっていると感じていました。一方でAIに任せようとすると、今度は睡眠中の音声や生活データを外部に送ることへの不安がつきまといます。

そこで Stellise では、ユーザーに我慢を強いるのではなく、「睡眠状態・天気・交通・予定」からAIが"今やるべきこと"を判断して寄り添う、人にやさしいアシスタントを目指しました。「起こす」だけで終わらず、起きたあとの段取りまで面倒を見る。そして、そのために必要なデータ解析は全部あなたの端末の中だけで完結させる——「賢いのに、安心して預けられる」朝の相棒を作りたい、という想いで開発しています。

## こだわったポイント
- **ストレスを与えない起こし方**: パズルや計算ではなく、眠りの浅いタイミングを狙って自然に起こし、起きたあとは「やるべきことの取捨選択」を提示するだけ。ユーザーは迷わず動くだけでいい体験を設計しました。
- **"思考の代行"**: 遅延や悪天候を検知すると、AIが「何を削り・何を先にやるか」を判断してタスクを自動で組み直す。朝の判断疲れそのものを肩代わりします。
- **プライバシー・バイ・デザイン**: 睡眠・音声解析はすべて端末内で完結させ、生データを外部送信しない／学習へ二次利用しない／広告も載せない。安心して枕元に置けることを最優先にしました。
- **世界観の作り込み**: 背景は画像素材に頼らず、時刻と天気からSceneKitでリアルタイム生成。朝・昼・夕・夜や晴れ／雨で空の表情が変わり、毎朝開くのが少し楽しみになる体験を狙いました。

## 技術的ハイライト
- **オンデバイス機械学習**: TensorFlow Lite（YAMNet）で環境音を分類し、CoreMotionの体動と組み合わせて睡眠の深さを推定。クラウド推論ゼロでスマートアラームを実現。
- **朝ルーティンの自動再計算**: 天気APIと経路情報（CoreLocation / MapKit）から出発時刻を算出し、遅延・悪天候時にタスク配分をAIが再構成。
- **生成的な3D背景（SceneKit）**: 空のグラデーション・太陽/月の弧・星・雲・窓を伝う雨滴までコードから生成し、時刻・天気に連動。
- **一貫したデザインシステム**: カラートークン・タイポ・ガラス表現を集約し、画面間で統一されたダークUIを担保。

## 使用技術
Swift / SwiftUI / Combine、SceneKit（背景の手続き生成）、TensorFlow Lite（YAMNet・オンデバイス音声分類）、CoreMotion、CoreLocation / MapKit、AVFoundation、App Intents / EventKit / UserNotifications、StoreKit、Firebase（Auth / Firestore）、天気・経路API

## 役割（個人開発）
個人開発。企画・UI/UXデザイン・実装すべてを担当。特にSceneKitによる天気連動3D背景の手続き生成と、オンデバイス睡眠解析（YAMNet＋CoreMotion）→スマートアラームのロジックを自前で設計・実装した。
